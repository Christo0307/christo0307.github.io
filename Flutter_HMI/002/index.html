第一，為何要呼叫runApp(MyApp())，這種間接的方式寫程式？是Flutter的慣例嗎？
這其實牽涉到：

Flutter 的 UI 架構 = Widget Tree

這是 Flutter 最核心的觀念之一。

runApp() 的作用是：

把某個 Widget 當成整個 App 的根節點(root)

而：

MyApp()

其實是：

建立一個 Widget 物件

Flutter 的世界觀

Flutter 認為：

整個畫面都是 Widget

包括：視窗、文字、按鈕、圖片、Layout、動畫

全部都是 Widget。

所以，App 本身也是 Widget。

因此，runApp(MyApp());

意思其實是：

「把 MyApp 這個 Widget 放到螢幕上」

runApp 是 Flutter Engine 提供的函式

它負責：

  1. 建立畫面
  2. 啟動 rendering engine
  3. 建立 Widget Tree
  4. 開始 UI loop

Widget Tree 是什麼？

這超重要。

Flutter UI 本質：

是一棵樹

例如：

MyApp
└── MaterialApp
└── DashboardPage
└── Scaffold
└── Column
├── Text
├── Text
└── Text

你現在真正該理解的核心

Flutter 不是：

「寫程式控制UI」

而是：

「用 Widget 描述UI」

這是最重要的思想轉變。

MyApp
App 入口

通常 Stateless。

class DashboardPage extends StatefulWidget
這是Widget class
負責描述：「這是一個會有 State 的 Widget」

class _DashboardPageState extends State
這是State class
負責：真正資料、邏輯、UI 更新

Widget 是描述
State 才是資料

簡單來說，這段程式碼的作用就是：「建立一個不需要改變狀態的 App 外殼（MyApp），使用 Google 設計風格（MaterialApp），把除錯標籤藏起來，並且一開機就直接顯示儀表板頁面（DashboardPage）。」

  1. 宣告元件類別
    Dart
    class MyApp extends StatelessWidget {
    class MyApp：宣告一個名叫 MyApp 的自訂類別，這通常是整個 Flutter App 的根元件（Root Widget）。

extends StatelessWidget：讓 MyApp 繼承自 StatelessWidget（無狀態微件）。這代表這個元件本身不需要動態管理或改變內部狀態，它的畫面一旦建立就不會自己改變。

  1. 覆寫父類別方法
    Dart
    @override
    @override：這是一個註解（Annotation），告訴編譯器底下的 build 方法是要覆寫父類別（StatelessWidget）裡原有的 build 方法，確保能執行我們自己定義的畫面邏輯。

  2. 建構畫面核心方法
    Dart
    Widget build(BuildContext context) {
    Widget build(…)：這是每一個 Widget 的核心方法，回傳值必須是一個 Widget。當 Flutter 要渲染這個元件時，就會呼叫這個方法。

BuildContext context：這是「上下文物件」，包含了這個 Widget 在整個元件樹（Widget Tree）中的位置資訊與 App 的全域主題等資料。

  1. 回傳 App 頂層外殼
    Dart
    return MaterialApp(
    return MaterialApp(：回傳一個 MaterialApp 元件。這是 Flutter 官方提供、遵循 Google Material Design 設計規範的應用程式外殼。它負責處理 App 的全域設定，例如：路由（轉場）、主題顏色、語系等。

  2. 設定首頁畫面
    Dart
    home: DashboardPage(),
    home:：這是 MaterialApp 的一個屬性，用來指定 App 啟動後第一個顯示的畫面（首頁）。

DashboardPage()：這裡呼叫了另一個自訂的 Widget（儀表板頁面），代表 App 一打開，使用者就會看到 DashboardPage 的內容。

  1. 隱藏除錯標籤
    Dart
    debugShowCheckedModeBanner: false,
    debugShowCheckedModeBanner: false：預設情況下，Flutter 在開發模式時，畫面的右上角會出現一個紅色的「DEBUG」斜角標籤。將這個屬性設為 false，就可以把那個除錯標籤隱藏，讓畫面看起來更乾淨。

這段程式碼定義了一個名為 DashboardPage 的 有狀態微件（StatefulWidget）。在 Flutter 中，當一個畫面的內容需要根據使用者的操作、網路請求或時間而動態改變時（例如：計數器、載入中的動畫、隨資料更新的圖表），就必須使用 StatefulWidget。

StatefulWidget 的特點是它一定會拆分成兩個類別（Class）來撰寫：一個是微件本身，另一個是負責管理狀態與畫面的 State 類別。你提供的這段就是第一個部分。

  1. 宣告StatefulWidget class

class DashboardPage：宣告一個名叫 DashboardPage 的自訂類別，通常代表你的儀表板主頁面。

extends StatefulWidget：讓它繼承自 StatefulWidget。這告訴 Flutter 這個元件是「有狀態」的，它的畫面在未來會隨著資料的變動而重新刷洗（Rebuild）。

  1. 綁定StatefulWidget的State class

createState()：這是 StatefulWidget 核心的方法。每當這個微件被放到畫面上的時候，Flutter 就會呼叫 createState() 來幫它配對一個負責管理資料與畫面的「大腦」（也就是 State）。

@override：在createState() 上方，實務上通常也會加上 @override 註解，代表它是覆寫父類別的方法。

下面的StatefulWidget的State class一定長這樣

當你呼叫 setState() 時，這個 State 類別就會重新執行 build 來更新畫面。

底線 _ 的含意：
在 Dart 語言中，類別名稱前面加上底線（如 _DashboardPageState），代表這個類別是私有的（Private）。這意味著這個大腦只服務 DashboardPage，其他的檔案是無法直接存取它的。這是一個很好的封裝實踐。

class _DashboardPageState …：定義專屬於 DashboardPage 的狀態類別。

int speed = 80;：定義一個整數變數，用來記錄車速，預設值是 80。

int rpm = 3200;：定義一個整數變數，用來記錄引擎轉速，預設值是 3200。

int fuel = 75;：定義一個整數變數，用來記錄剩餘油量百分比，預設值是 75。

註：這三個變數就是這個頁面的「狀態（State）」，未來如果呼叫 setState() 改變它們，畫面上的數字就會跟著變。

return Scaffold(…)：Scaffold 是 Flutter 內建的頁面鷹架。它提供了一個標準的頁面結構（可以放導覽列、側邊欄、主體等）。

backgroundColor: Colors.black：將整個頁面的背景顏色設定為黑色，這樣能讓儀表板的數據看起來更顯眼（有科技感）。

版面配置：置中與垂直排列 (Center & Column)

body:：設定 Scaffold 的主體內容。

Center(…)：一個版面配置元件，會把它裡面的子元件（child）拉到畫面的正中央。

Column(…)：一個垂直排列的佈局元件，會把它裡面的所有子元件（children）由上到下一列一列排好。

mainAxisAlignment: MainAxisAlignment.center：讓 Column 內的所有子元件在垂直方向上也居中對齊。

children: […]：裡面放的是一個陣列（List），用來放置所有要垂直排列的元件。

  1. 顯示時速 (SPEED)

Text(…)：用來顯示文字的元件。

“SPEED: $speed km/h”：這裡用了 Dart 的字串插值（String Interpolation）。$speed 會被自動替換成上面定義的變數值，所以畫面上會顯示 SPEED: 80 km/h。

style: TextStyle(…)：設定文字的樣式。

color: Colors.green：字體顏色設為綠色。

fontSize: 40：字體大小設為 40（非常大且清晰）。

  1. 區隔區塊 (SizedBox)

SizedBox(height: 20)：這是一個隱形的盒子。在這裡的作用是充當間隔（Margin），在時速和轉速之間空出 20 個邏輯像素的垂直距離，避免文字擠在一起。
