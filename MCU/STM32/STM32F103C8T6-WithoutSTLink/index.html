不使用ST Link，改用CH340，燒錄STM32韌體

(Jan. 16th 2026)

早上工作在趕案子的時候，ST Link一直連不上開發板。

出現以下的錯誤畫面：

Can not connect to target!
Please select “Connect Under Reset” mode from Target->Settings menu
and try again.
If you’re trying to connect to a low frequency application, please select
a lower SWD Frequency mode from Target->Settings menu.

查了一下資料，其實不需要ST Link，也有可以燒錄韌體的方法。

一、接線，BOOT1=0，BOOT0=1

在STM32F10xxx的開發板，可以透過設定BOOT 0和BOOT 1腳位的方式，

進入燒錄韌體的模式。

如規格書所示，設定BOOT1=0，BOOT0=1，即可燒錄韌體。

照以下的圖示，連接CH340和開發板

STM32F103C8T6開發板的接線圖如下

點擊開啟影片，把BOOT0和BOOT1腳位，分別接到VCC和GND

二、編寫韌體

接著，我們把韌體程式寫好之後，(例如我寫了一個A6 GPIO點亮LED的韌體)

在專案上點右鍵→Properties

在專案設定，點選C/C++ Build→Settings→MCU/MPU Post build outputs

打勾Convert to binary file (-O binary)

打勾Convert to Intel Hex file (-O ihex)

接著回到原來的視窗，建置方案Build

這時，就會產生可以給Flymcu燒錄的hex韌體檔

三、下載FLYMCU

到單片機在線編程網

https://www.mcuisp.com/

軟件下載→Free STM32 ISP software→點擊下載

https://www.mcuisp.com/chinese%20mcuisp%20web/ruanjianxiazai-chinese.htm

下載FLYMCU程式

四、使用FLYMCU燒錄韌體

開啟FlyMcu.exe

在FlyMcu的主畫面，應該會看到偵測到CH340的COM Port

如果沒出現，按一下Enum Port或是關掉FlyMcu，先插CH340，再開啟FlyMcu

接著選取剛剛編譯好的hex檔

檔案預設應該在

C:\Users\你的名字\STM32CubeIDE\workspace_1.18.0\你的專案資料夾\Debug

然後按Read ChipInfo( R)

右邊應該會出現晶片的資訊，代表已經進入可以燒錄韌體的模式

接著按Start ISP(P)，開始燒錄韌體

五、成果

燒錄完韌體之後，先關掉程式

然後把BOOT0接到GND

接著重新啟動MCU

可以看到新燒錄的韌體已經開始執行
